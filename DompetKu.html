Dokumen Projek DompetKu

Fasa A: Reka Bentuk

Definisi Masalah: Pengguna (terutama pelajar) sering hilang jejak perbelanjaan harian mereka, menyebabkan masalah kewangan menjelang hujung bulan.

Sasaran Pengguna: Pelajar kolej, universiti dan golongan belia.

Kerangka Wayar (Wireframes): Sistem disediakan dengan 4 skrin berbeza:

  • Utama: Paparan baki, bar peratusan bajet & transaksi terkini.
  • Senarai: Rekod penuh sejarah perbelanjaan (boleh diskrol).
  • Tambah: Borang merekod butiran dan jumlah wang.
  • Tetapan: Konfigurasi bajet bulanan dan memaparkan dokumen projek.

Pemilihan Seni Bina: Single Page Application (SPA). Pendekatan ini dipilih kerana ia memberi pengalaman interaktif yang lancar (tanpa muat semula halaman pelayar), mesra data, dan amat bersesuaian dengan format aplikasi mudah alih.

Fasa B: Pemilihan Alatan & Strategi Prompting

Justifikasi Alatan: Ejen AI (sebagai pembantu) dan penyunting web dipilih untuk menjana, menstruktur serta mempercepat proses pemformatan UI / kod responsif.

Strategi Prompting:

  • Awal: "Bina aplikasi web HTML/JS SPA untuk urus perbelanjaan bajet, ada 4 tab antaramuka."
  • Penambahbaikan: "Jadikan ia sistem bernama DompetKu mengikut fasa tertentu. Tambah fungsi Imej (logo), peratusan bajet, buang alert browser dan guna modal."

Fasa C: Pembinaan

Ciri-ciri sistem merangkumi:

  • Aliran Navigasi: Lantaran antara 4 skrin utama diuruskan oleh fungsi JS DOM tanpa sebarang ralat atau muat semula (reload).
  • Logik Aplikasi: Pengiraan baki secara matematik, peratusan penggunaan bajet, dan menapis penyusunan senarai rekod terkini.
  • Komponen Antarmuka: Telah mengaplikasikan penggunaan Butang, Input Teks (borang), Dropdown (Select), dan Imej (Logo SVG DompetKu di bahagian atas).
  • Komponen Lanjutan (Pangkalan Data): Sistem ini memanfaatkan API Web LocalStorage bertindak sebagai Pangkalan Data sisi-klien untuk menyimpan rekod pengguna secara persisten.

Fasa D: Pengujian Fungsian

Ujian fungsian ini disediakan di dalam sistem untuk meniru proses log ujian mandatori (merekodkan input ujian, jangkaan keputusan dan hasil sebenar). Klik butang di bawah untuk menjalankan pengujian log automatik.
